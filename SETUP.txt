Talib-Awn — Full Local Setup Guide
=====================================
Follow every step in order. About 10 minutes total.


FILE STRUCTURE
--------------
Put ALL files in ONE folder, for example:  C:\talib-awn\  or  ~/talib-awn/

  talib-awn/
  ├── app.py                   Flask entry point
  ├── routes.py                All API endpoints
  ├── models.py                Database models
  ├── parsers.py               University news scraper
  ├── requirements.txt         Python packages list
  │
  ├── style.css                Shared styles (blue brand palette)
  ├── auth.css                 Auth page styles
  ├── api.js                   Shared fetch + token helpers
  ├── chat-widget.js           Floating chat bubble (drops into any page)
  │
  ├── index.html               Landing page (public)
  ├── login.html               Sign in
  ├── register.html            Create account + OTP
  ├── forgot-password.html     Reset password (3-step)
  ├── dashboard.html           Smart redirect by role
  ├── student-dashboard.html   Member home
  ├── employer-dashboard.html  Employer home
  ├── admin-panel.html         Admin home
  └── chat.html                Full messages page

  Optional:
  └── logo.png                 Drop your Talib-Awn logo PNG here


STEP 1 — Install requirements
------------------------------
Python 3.10+  →  https://www.python.org/downloads/
MySQL 8.0+    →  https://dev.mysql.com/downloads/installer/

Verify:
  python --version     should say 3.10 or higher


STEP 2 — Create the MySQL database (one time only)
---------------------------------------------------
Open a terminal and run:

  mysql -u root -p

Press Enter if you have no password.
Inside the MySQL shell:

  CREATE DATABASE club_chlef CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
  EXIT;

Tables are created automatically when the app starts.


STEP 3 — Change DB credentials if needed
-----------------------------------------
Open app.py and find:
  'mysql+pymysql://root:@localhost/club_chlef'

Format: mysql+pymysql://USERNAME:PASSWORD@HOST/DATABASE

  root, no password  →  mysql+pymysql://root:@localhost/club_chlef
  root, pw = 123     →  mysql+pymysql://root:123@localhost/club_chlef


STEP 4 — Install Python packages
----------------------------------
  cd path/to/talib-awn
  pip install -r requirements.txt


STEP 5 — Start the backend
-----------------------------
  python app.py

You should see:
  [db] Tables checked / created.
   * Running on http://0.0.0.0:5000

Leave this terminal open the whole time.


STEP 6 — Open the frontend
----------------------------
Double-click login.html, or open it in Chrome/Firefox.
The browser URL will look like:  file:///C:/talib-awn/login.html


STEP 7 — Add your logo (optional)
------------------------------------
Save the Talib-Awn logo PNG as  logo.png  in the same folder.
It will show automatically in all nav bars and sidebars.
If logo.png is missing, the letter "ط" shows as fallback.


STEP 8 — Create the first Admin account
-----------------------------------------
Method A — Register page (easiest):
  1. Open register.html
  2. Click the "Admin" tab
  3. Fill in details, get OTP email, verify it
  4. Login → lands on admin-panel.html

Method B — Direct API (if email not working yet):

  On Mac/Linux:
    curl -X POST http://localhost:5000/api/v1/admin/register \
      -H "Content-Type: application/json" \
      -d '{"firstname":"Admin","lastname":"Awn","email":"admin@talib.dz","phone":"0550000001","password":"admin123"}'

  On Windows PowerShell:
    Invoke-WebRequest -Uri "http://localhost:5000/api/v1/admin/register" `
      -Method POST -ContentType "application/json" `
      -Body '{"firstname":"Admin","lastname":"Awn","email":"admin@talib.dz","phone":"0550000001","password":"admin123"}'


STEP 9 — Quick API health check
---------------------------------
Open these in your browser while the server is running:

  http://localhost:5000/api/v1/client/grades     returns list of grades
  http://localhost:5000/api/v1/client/domains    returns list of domains
  http://localhost:5000/api/v1/stats             returns project/event counts


CHAT SYSTEM
-----------
Two modes:

1. FLOATING WIDGET (available on every dashboard)
   - Blue chat button, bottom-right corner of the screen
   - Shows unread message count
   - Click to open a mini chat panel with recent conversations
   - "Full view →" opens chat.html
   - Powered by: chat-widget.js (already injected into all dashboards)

2. FULL CHAT PAGE (chat.html)
   - Two-panel layout: conversation list + message view
   - Message bubbles with timestamps and read receipts (checkmarks)
   - Job context bar: link a job to the conversation
   - Send Offer button: sends a structured job offer card in the chat
   - Receiver can Accept or Decline the offer directly in chat
   - Quick reply buttons for common phrases
   - Typing indicator animation
   - Mobile responsive (conversation list slides away)
   - Attach file button (UI only, wire to your storage backend)

Note: Chat currently uses in-memory demo data. To save messages permanently,
add a Message model to models.py and POST/GET endpoints to routes.py,
then replace the sendMessage() call in chat.html to use apiFetch().

API REFERENCE
-------------
Base URL: http://localhost:5000/api/v1

No token needed:
  POST  /client/pre-register       Step 1 registration (sends OTP)
  POST  /client/verify-otp         Step 2 (creates account)
  POST  /client/login              Login → returns access + refresh tokens
  POST  /client/forgot-password    Step 1 reset
  POST  /client/verify-reset-otp   Step 2 reset
  POST  /client/reset-password     Step 3 reset
  GET   /client/grades             Valid grades list
  GET   /client/domains            Valid domains list
  GET   /stats                     Public project/event count
  (same endpoints exist under /admin/ prefix for admin accounts)

Protected (send Authorization: Bearer <token>):
  GET   /me                        Your profile
  PUT   /me                        Update profile
  GET   /token/refresh             Refresh access token
  GET   /events                    All events
  POST  /events/<id>/join          Join event
  DELETE /events/<id>/quit         Leave event
  GET   /projects                  All projects
  POST  /projects/<id>/like        Toggle like
  GET   /projects/<id>/comments    Get comments
  POST  /projects/<id>/comments    Add comment
  DELETE /projects/<id>/comments/<id>   Delete comment
  POST  /users/<id>/follow         Follow/unfollow user
  POST  /search                    Search members by name
  POST  /info                      Get user profile + projects by ID
  GET   /users/<id>/followers      User's followers
  GET   /users/<id>/following      User's following list
  GET   /getinfo                   University news list
  GET   /getsuperinfo/<id>         Article + optional ?lang=fr translation

Admin only:
  GET   /admin/fetch-all           All members
  POST  /admin/invite              Send invite email
  POST  /admin/ban                 Ban user
  POST  /admin/unban               Unban user
  POST  /events                    Create event (auto-emails all members)
  PUT   /events/<id>               Edit event
  DELETE /events/<id>              Delete event
  POST  /announcements             Create announcement
  PUT   /announcements/<id>        Edit announcement
  DELETE /announcements/<id>       Delete announcement


TROUBLESHOOTING
---------------
"Can't connect to MySQL"
  MySQL is not running.
  Windows: open Services app and start MySQL
  Linux/Mac: sudo service mysql start

"Access denied for user root"
  Wrong password. Fix the connection string in app.py (Step 3).

"No module named flask"
  Run: python -m pip install -r requirements.txt

"Address in use" on port 5000
  Change 5000 to 5001 in app.py AND in api.js line 1.

OTP not arriving
  Add a print to routes.py inside _pre_register() after the otp= line:
    print(f"[DEV] OTP for {email}: {otp}")
  Then check your terminal window.

Chat widget not showing
  Make sure chat-widget.js is in the same folder as the HTML files.
  Open browser DevTools console and check for 404 errors.

CORS error in browser console
  Backend is not running, or running on a different port.
  Check python app.py is active. Make sure api.js line 1 matches.
